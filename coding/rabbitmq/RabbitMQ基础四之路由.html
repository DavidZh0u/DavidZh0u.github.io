    # RabbitMQ基础四之路由


    # 场景假设

    前面我们实现了消息的发布与订阅。它就像是消息的群发，所有绑定在交换机上的队列(临时队列断开连 接就取消绑定了)都能收到消息。现在我们有这么一个场景，我们的应用有三种日志：info、warning、 error。现在希望这三种日志消息，通过一个交换机，投递到不同日志处理队列里去。两个队列，一个是 普通队列，用作error日志的报警处理与记录；一个临时队列，用作全体日志的可视化展现。这样做，也是 考虑到日志的可靠性，因为临时队列断线就消失了，而普通队列只要绑定到交换机，即使客户端崩溃也不影 响消息的保存。因为消息服务端有持久化保证，这个我们在后面会去介绍它。
    ❶问题
    前一个发布/订阅实现，我们是使用的扇出交换机(FanoutExchange)，它就像波纹一样在扇区扩散到每一个绑定的 队列上去。我们在使用这种交换机的时候，没有写routingKey的值，同样的我们绑定队列到这个交换机上 也没有写routingKey。问题是，它是通过默认的""(null)routingKey绑定的，还是扇出交换机不处理routingKey？ 我们后面验证下。
    ❷DirectExchange
    这种交换机处理routingKey,队列与交换机之间多了一层routingKey的导向。生产的消息必须携带routingKey，然后 根据routingKey匹配队列去完成消息的投递。下面就用这种交换机实现我们上面的场景。

    # 验证

    ❶验证FanoutExchange是否对路由键有效？
    配置类新增FanoutExchange，上一篇我们把扇出交换机取名：psExchange。这里 我们直接首字母小写，定义名为fanoutExchange的扇出交换机。

        @Bean
    public FanoutExchange fanoutExchange(){
        return new FanoutExchange("fanoutExchange");
    }

    在生产者和消费者都配置这个交换机，然后本次试验代码，所有绑定到directExchange 交换机上的队列，全部改写到绑定fanoutExchange。这时出现了一个语法错误，此时根本 没有with(routingKey)。这里也就告诉我们，扇出交换机设计为不处理路由键。直接看 源码：如下。当绑定FanoutExchange交换机时，执行到to(Exchange)，如果Exchange是 FanoutExchange类型，直接就返回Binding，完成了Bean配置工作。然而Exchange是 DirectExchange类型，就返回的不是Binding，而是DirectExchangeRoutingKeyConfigurer。 继续跟进这个DirectExchangeRoutingKeyConfigurer，发现是它提供了一个with(routingKey) 的方法，这个方法才是返回Binding关系的。到这里还能知道一个事，如果你是将队列绑定到 DirectExchange交换机上，不使用路由键行吗？答案显然是不行，源码已经明白说了，你不设置 路由键，DirectExchange类型绑定执行到to()方法，你还是无法得到Binding,只有继续执行 with(routingKey),才能正确绑定。而with(""),也算一种路由键，后台页面能看到这种绑定 关系。这种routingKey=""我已经测试，生产方也发带这个路由键的消息即可。绑定关系如下图。 

    
	/**
	 * General destination configurer.
	 */
	public static final class DestinationConfigurer {

		protected final String name; // NOSONAR

		protected final DestinationType type; // NOSONAR

		DestinationConfigurer(String name, DestinationType type) {
			this.name = name;
			this.type = type;
		}

        //扇出交换机到这里已经确定了绑定关系
		public Binding to(FanoutExchange exchange) {
			return new Binding(this.name, this.type, exchange.getName(), "", new HashMap<String, Object>());
		}

		public HeadersExchangeMapConfigurer to(HeadersExchange exchange) {
			return new HeadersExchangeMapConfigurer(this, exchange);
		}
		
        //direct交换机到还没有确定绑定关系
		public DirectExchangeRoutingKeyConfigurer to(DirectExchange exchange) {
			return new DirectExchangeRoutingKeyConfigurer(this, exchange);
		}

		public TopicExchangeRoutingKeyConfigurer to(TopicExchange exchange) {
			return new TopicExchangeRoutingKeyConfigurer(this, exchange);
		}

		public GenericExchangeRoutingKeyConfigurer to(Exchange exchange) {
			return new GenericExchangeRoutingKeyConfigurer(this, exchange);
		}
	}
	
    /**
     * Direct exchange routing key configurer.
     */
    public static final class DirectExchangeRoutingKeyConfigurer extends AbstractRoutingKeyConfigurer {

        DirectExchangeRoutingKeyConfigurer(DestinationConfigurer destination, DirectExchange exchange) {
            super(destination, exchange.getName());
        }
        
        //direct交换机在这里确定绑定关系
        public Binding with(String routingKey) {
            return new Binding(destination.name, destination.type, exchange, routingKey,
                    Collections.<String, Object>emptyMap());
        }

        public Binding with(Enum<?> routingKeyEnum) {
            return new Binding(destination.name, destination.type, exchange, routingKeyEnum.toString(),
                    Collections.<String, Object>emptyMap());
        }

        public Binding withQueueName() {
            return new Binding(destination.name, destination.type, exchange, destination.name,
                    Collections.<String, Object>emptyMap());
        }
    }
