    # RabbitMQ基础之前置环境

    # 首先安装好rabbitmq

    window环境下安装比较简单，查一下资料即可。
    Linux-centos7环境的安装，可以看 Centos7环境安装rabbitmq

    # 浏览rabbitmq后台管理页面

    rabbitmq默认管理员账号和密码
    quest/quest

    # 新建开发用户名和密码

    依次填写账户名，密码，重复密码。
    Tags:这个选择Admin
    点击[Add user]按钮完成
    然后上面出现一个新的账号，用这个账号可以访问后台管理页面。

    # 新建Virtual Hosts

    Virtual Hosts可以看做是mysql中一个库。绑定在这个库中的消息，与其他库是隔离的。 不同的系统用各自的库，彼此不干扰。
    点击右侧的Virtual Hosts标签，跳转到新页面。
    Name：一般以“/”开头，我这里的就是“/dev”
    Description: 描述，就解释下你这个干啥用的。
    Tags:可以不写。
    点击[Add virtual host]按钮完成
    当前登录账号为：guest. 你此时创建的V-host绑定的账号就是guest。

    # 绑定vhosts和新建账号

    在v-host页中name一栏中，点击你新创建的v-host,比如我的就是“/dev”。 点击它，跳转到新页面。你可以看到当前绑定的账号是guest。如果你需要绑定新建账号，如下图中。选择账号，点击[Set permission]。然后新建账号和新建V-host就绑定在一起了。绑定其他账号也是如此操作。

    # 前置工作完结

    这里我们已经做好了客户端开发的前置工作，后面我们将写代码运用一下消息队列，实现消息的生产与消费。
